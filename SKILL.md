# Анализ рисков регионов: Матрица Риск × Рейтинг РУСАДА

## Контекст и аудитория

Самостоятельный аналитический модуль, который **дополняет** основной проект (ipynb), но не заменяет его. Основной проект содержит собственные зоны риска — данный скилл добавляет к ним рейтинговый срез РУСАДА по субъектам РФ. Целевая аудитория — работники антидопинговой сферы. Объект анализа — **регионы (субъекты РФ)**, не виды спорта. Выдача — дополнительный вход для управленческих решений по матрице приоритетов.

**Важно:** xlsx содержит дублирующуюся строку-заголовок на строке 50 — фильтровать строки, где `Субъект РФ == "Субъект РФ"` или `ФО == "ФО"`.

## Зоны риска

Прогнозная ipynb-модель для регионов отсутствует — зона определяется по итоговому баллу.

В логике и данных — только текстовые коды. Эмодзи допустимы исключительно как декор в Excel и дашборде.

```
RED    — итого < 95   (<50% от 190)    (критический уровень)
ORANGE — 95 ≤ итого < 130              (умеренный уровень)
GREEN  — итого ≥ 130                   (низкий риск)
```

Порядок проверки: RED → ORANGE → GREEN. Регион получает **ровно одну** зону.

Соответствие терминов матрицы:
- «Систематичность нарушений» = RED или ORANGE
- «Нет систематичности» = GREEN

## Матрица приоритетов (порог рейтинга = 130 баллов)

| Приоритет | Квадрант | Рекомендация |
|---|---|---|
| **1** | Рейтинг < 130 + RED/ORANGE | Срочное вмешательство РУСАДА; приоритизировать работу по невыполненным критериям всех трёх блоков |
| **2** | Рейтинг ≥ 130 + RED/ORANGE | Усилить антидопинговое образование; беседы с ответственными; рассмотреть внесоревновательный контроль |
| **3** | Рейтинг < 130 + GREEN | Продолжать профилактику; приоритизировать невыполненные критерии рейтинга; превентивная работа с регионом |
| **4** | Рейтинг ≥ 130 + GREEN | Поддерживать текущий уровень; обмен лучшими практиками; плановый мониторинг |

## Алгоритм выполнения

### Этап 0 — Изучение входных данных (обязателен)

Перед анализом вывести артефакты изучения. Переходить к Шагу 1 только после их вывода.

**Источник — xlsx рейтинга:**
1. Число листов и структура активного листа (колонки, строки).
2. **N_рейтинг** — итоговое число регионов (после фильтрации строк-заголовков).
3. Список колонок и соответствие блокам критериев.
4. Первые 3 и последние 3 региона по списку.
5. Логика начисления баллов по блокам в 2–3 предложениях.

Если число регионов не совпадает с ожидаемым (~89) — найти и задокументировать причину. Продолжать с неполной таблицей запрещено.

### Шаг 1 — Скачать xlsx

### Шаг 2 — Извлечение данных рейтинга

По каждому из N_рейтинг регионов зафиксировать:
- выполненные критерии по каждому блоку (балл присвоен),
- невыполненные критерии (балл отсутствует),
- итоговый балл и место.

Отфильтровать строки-заголовки: `Субъект РФ == "Субъект РФ"` или `ФО == "ФО"`.

### Шаг 3 — Зоны риска и квадранты

Определить зону (RED/ORANGE/GREEN) по порогам из раздела «Зоны риска». Присвоить приоритет 1–4 по матрице выше.

**Краевые случаи:**
- Итоговый балл = 0 → зона RED, не удалять, обоснование: «нулевой балл».
- Отсутствуют данные по отдельному блоку → зону определять по доступным блокам, в обосновании указать «данные по блоку X отсутствуют».
- 0 выполненных или 0 невыполненных критериев → писать «нет», ячейку пустой не оставлять.

**Инвариант полноты:** число строк итоговой таблицы = N_рейтинг. Вывести это равенство с числами.

### Шаг 4 — Рекомендации и обоснования

Рекомендация — дословно из таблицы приоритетов выше.

Обоснование:
- RED/ORANGE: «[Зона] риск (итого: [балл] из 190, [%]%); нехватка баллов по критериям: [перечислить невыполненные]» (если все выполнены — «критерии рейтинга выполнены полностью»).
- GREEN: «Низкий риск (итого: [балл] из 190, [%]%); [недостающие критерии или "все критерии выполнены"]».

### Шаг 5 — Самопроверка (обязательна перед выдачей)

Вывести каждый инвариант с фактическими числами:

1. Число строк итоговой таблицы = N_рейтинг.
2. Сумма строк по четырём приоритетам = число строк итоговой таблицы.
3. Каждая строка имеет ровно одну зону из {RED, ORANGE, GREEN} и ровно один приоритет 1–4.
4. Нет строк RED/ORANGE с приоритетами 3–4; нет строк GREEN с приоритетами 1–2.
5. Нет строк с рейтингом ≥ 130 в приоритетах 1 и 3; нет строк с рейтингом < 130 в приоритетах 2 и 4.
6. Распределение строк по зонам (RED/ORANGE/GREEN — по сколько). Если зона содержит 0 строк — взять 3 случайные строки и вручную проверить пороги.
7. Нет строк с `Субъект РФ == "Субъект РФ"` (строки-заголовки отфильтрованы).
8. Все строки содержат официальное название субъекта РФ (не сокращение).
9. Итоговая таблица отсортирована: приоритет по возрастанию → RED раньше ORANGE → балл по убыванию.
10. На дашборде точки не перекрываются (jitter применён), подписи читаемы.

Если хотя бы один инвариант нарушен — исправить и повторить самопроверку. Выдавать результат с нарушенным инвариантом запрещено.

## Стиль визуализации

```python
INK  = "#0F2D52"; SUB = "#7C8DA6"; GRID = "#EEF2F7"
FONT = "Inter, Segoe UI, Roboto, Arial, sans-serif"

# Для бар-графика и Excel (3 зоны):
ZONE_COLORS = {"RED": "#DC2626", "ORANGE": "#F59E0B", "GREEN": "#10B981"}

# Для матрицы квадрантов scatter (4 приоритета):
QUADRANT_COLORS = {
    "Приоритет 1 (низкий рейтинг + систематичность)": "#DC2626",
    "Приоритет 2 (высокий рейтинг + систематичность)": "#F59E0B",
    "Приоритет 3 (низкий рейтинг + нет систематичности)": "#3B82F6",
    "Приоритет 4 (высокий рейтинг + нет систематичности)": "#10B981",
}
# Правило: scatter красится по QUADRANT_COLORS, бар и Excel — по ZONE_COLORS. Не смешивать.

# Scatter: jitter ±0.25 по Y, marker size≈10, opacity≈0.82
# Горизонтальные бары: marker_cornerradius=5, textposition="outside", cliponaxis=False
```
